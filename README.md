# Consultas SQL y Filtros Avanzados

El objetivo principal de este ejercicio práctico es aplicar consultas `SELECT` utilizando filtros condicionales (`WHERE`), operadores lógicos (`AND`, `OR`, `NOT`) y coincidencia de patrones (`LIKE`, `ILIKE`) en **PostgreSQL**.

## Estructura del Proyecto

El proyecto está organizado de la siguiente manera:

```text
M5_EP1/
├── img/                       # Evidencias de ejecución
│   ├── M5_EP1_Evidencia01.png
│   ├── ...
│   └── M5_EP1_Evidencia08.png
│
├── datosdeprueba.sql          # Script que crea la tabla 'clientes' y carga datos de prueba
└── M5_EP1.sql                 # Script con las 8 consultas de solución al ejercicio
```

## Tecnologías Utilizadas

1. **Motor de Base de Datos:** PostgreSQL

2. **Herramienta de Gestión:** pgAdmin 4

3. **Lenguaje:** SQL

## Instrucciones de Ejecución

Para probar este proyecto en tu entorno local (pgAdmin 4):

1. **Preparación del entorno:**

- Abre el archivo `datosdeprueba.sql` en tu Query Tool.

- Ejecuta el script completo para crear la tabla clientes y poblarla con los datos iniciales.

2. **Ejecución de consultas:**

- Abre el archivo `M5_EP1.sql`.

- Ejecuta las consultas una por una para verificar los resultados solicitados en la guía de ejercicios.

## Evidencias

Las capturas de pantalla que validan el funcionamiento de cada consulta se encuentran almacenadas en la carpeta `img/`, numeradas del 01 al 08 correspondiendo a cada requerimiento.
